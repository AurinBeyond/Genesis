# OUTPUT STANDARD v1.0 (LOCKED)

## PURPOSE

This document defines how all outputs must be structured and validated.

All agents must follow this standard.

---

## I. OUTPUT STRUCTURE (MANDATORY)

Every output must follow this structure unless explicitly defined otherwise.

## TITLE

## INTRO (2–4 sentences)

## MODULE 1 — CONCEPT  
Explain the idea clearly and simply.

## MODULE 2 — PRACTICE  
Describe what the user can do.

## MODULE 3 — PROTOCOL  
Provide a simple, repeatable structure or method.

## FINAL NOTE  
Short closing that reinforces clarity (not emotion).

---

## II. LANGUAGE RULES

- No hype  
- No urgency  
- No exaggerated claims  
- No manipulation  
- No “life-changing” promises  

Use:

- clear  
- direct  
- calm  
- structured language  

---

## III. CLARITY RULE

If a sentence can be simplified → simplify it.

Avoid:
- complex wording  
- abstract phrasing without meaning  
- unnecessary length  

---

## IV. USER ACTION PRINCIPLE

Every output must include:

- a clear next step  
OR  
- a clear reflection trigger  

If neither exists → output is incomplete.

---

## V. VALIDATION RULE

Output is considered VALID only if:

- follows structure  
- follows SYSTEM-INPUT.md  
- follows GENESIS-LAWS.md  
- follows GENESIS-OPERATION.md  
- requires minimal manual correction  
- provides clear user value  

---

## VI. FAILURE CONDITION

Output is FAIL if:

- sounds like advertising  
- contains hype or emotional pressure  
- includes unsafe or unverifiable claims  
- is vague or unusable  

---

## VII. SELF-CORRECTION RULE

If output fails:

- agent must rewrite  
- agent must simplify  
- agent must revalidate  

User must NOT fix agent output repeatedly.

---

## VIII. CONSISTENCY RULE

All outputs must:

- match brand tone  
- match structure  
- match system logic  

No random style changes allowed.

---

## IX. OUTPUT TYPE AWARENESS

Not all outputs are identical.

Agents must adapt structure based on output type:

- Long-form → follow full structure  
- Short-form → compress modules  
- Instruction → emphasize clarity and steps  
- Insight → emphasize reflection and understanding  

Structure must remain recognizable, but not rigid.

---

## CORE PRINCIPLE

Clear > clever  
Simple > impressive  
Usable > perfect
